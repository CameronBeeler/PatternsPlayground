The use of Generics, and especially the use of reference variables to inherited objects is interesting.
It seems to me that as developers began to see the possibilities of utilizing multiple inherited objects
to 'build' an object piece-meal, that they ran into challenges with accessing the right object reference...
as such, the need for generics and the need for referencing extended objects through overriden methods...

As Dmitry continues to 'evolve' code to enable additional functionality, he is actually showing us the
progression of the requirements for JAVA and how those requirements were met...
